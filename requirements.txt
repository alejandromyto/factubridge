# Política de versionado:
# - Librerías críticas de seguridad/AEAT (cryptography, signxml, lxml, xmlschema, asyncpg, qrcode) se fijan con ==
#   para garantizar compatibilidad exacta y cumplimiento normativo.
# - Frameworks y librerías core (FastAPI, SQLAlchemy, Pydantic, Celery, HTTPX, Alembic, Zeep, Sentry) usan rangos >=X,<major+1
#   siguiendo semver, permitiendo actualizaciones menores y parches sin romper compatibilidad.
# - Herramientas de desarrollo, testing y linting (pytest, black, flake8, mypy, etc.) se dejan con >=X
#   para flexibilidad en entorno de desarrollo.
# - Desarrollo: requirements.txt con rangos semánticos (>=X,<major+1) y == solo en librerías críticas (seguridad/AEAT/drivers).
# - Producción: requirements.lock (generar con `pip freeze`) para reproducibilidad exacta, incluyendo sub-dependencias.
# - Flujo: actualizar versiones en requirements.txt, validar en CI/staging, luego `pip freeze > requirements.lock` y desplegar.
